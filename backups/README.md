# 自动备份功能说明

## 功能概述

本系统已实现自动备份功能，可在管理员后台设置备份频率、时间和保留数量。系统会按照设定的时间自动执行数据库备份，并清理超出保留数量的旧备份。

## 主要功能

1. **自动备份设置**：在管理员后台的系统设置页面，可以配置：
   - 自动备份开关（启用/禁用）
   - 备份频率（每天/每周/每月）
   - 备份时间（24小时制）
   - 保留备份数量（1-30个）

2. **自动执行备份**：系统会按照设定的时间自动执行备份，无需手动操作。

3. **备份文件管理**：系统会自动清理超出保留数量的旧备份，保持备份目录的整洁。

4. **备份文件存储**：备份文件存储在 `backups/` 目录下，文件名格式为 `backup_YYYYMMDD_HHMMSS.sql`。

## 技术实现

### 核心文件

1. **`admin/admin.js`**：管理员后台系统设置页面，添加了自动备份设置选项。

2. **`api/auto-backup.php`**：自动备份脚本，负责执行备份操作。

3. **`api/test-auto-backup.php`**：测试脚本，用于手动触发备份。

4. **`api/init-settings-table.php`**：初始化系统设置表。

### 定时执行

要实现定时自动备份，需要在服务器上设置CRON作业：

#### Linux系统

在终端执行：
```bash
crontab -e
```

添加以下行（每天凌晨2点执行）：
```
0 2 * * * php /path/to/jg/api/auto-backup.php
```

#### Windows系统

1. 打开任务计划程序
2. 创建基本任务
3. 设置触发时间（每天凌晨2点）
4. 操作选择"启动程序"
5. 程序路径选择PHP可执行文件，参数为自动备份脚本路径

## 使用方法

### 1. 初始化系统设置

首次使用自动备份功能前，需要初始化系统设置表：

```bash
php api/init-settings-table.php
```

### 2. 配置自动备份设置

1. 登录管理员后台
2. 进入"系统设置"页面
3. 在"自动备份设置"部分配置：
   - 启用自动备份
   - 选择备份频率（每周）
   - 设置备份时间（如02:00）
   - 设置保留备份数量（2个）
4. 点击"保存设置"

### 3. 测试自动备份

可以使用测试脚本手动触发备份：

```bash
php api/test-auto-backup.php
```

### 4. 查看备份文件

备份文件存储在 `backups/` 目录下，可以通过FTP或文件管理器查看。

## 备份文件恢复

要恢复备份，只需将备份文件中的SQL语句导入到数据库即可。

## 注意事项

1. 确保 `backups/` 目录有写入权限
2. 确保 `logs/` 目录有写入权限（用于存储备份日志）
3. 定期检查备份是否正常执行
4. 建议将备份文件下载到本地保存，以提高数据安全性

## 故障排除

### 备份失败

1. 检查数据库连接是否正常
2. 检查 `backups/` 目录权限
3. 查看 `logs/auto-backup.log` 日志文件获取详细错误信息

### 自动备份不执行

1. 检查CRON作业是否正确设置
2. 检查系统时间是否准确
3. 检查自动备份设置是否启用

### 备份文件过大

如果备份文件过大，可能是数据量较大，可以考虑：
1. 增加服务器存储空间
2. 调整备份频率
3. 定期清理不必要的数据
